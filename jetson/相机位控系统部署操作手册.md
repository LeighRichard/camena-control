## 3. TensorRT 安装与配置

### 3.1 检查 TensorRT 安装状态

```bash
# 检查 TensorRT 包
dpkg -l | grep tensorrt

# 预期输出应包含:
# ii  libnvinfer8      8.x.x
# ii  libnvinfer-dev   8.x.x
# ii  libnvinfer-plugin8  8.x.x
```

### 3.2 安装 TensorRT Python 绑定

**这是最关键的一步！**

```bash
# 更新软件源
sudo apt-get update

# 安装 TensorRT Python 绑定
# 注意：JetPack 4.6 中某些包名称不同，请按以下顺序安装

# 首先安装核心包
sudo apt-get install -y \
    python3-libnvinfer \
    python3-libnvinfer-dev \
    libnvinfer-bin \
    libnvinfer-dev \
    libnvinfer-plugin8

# 尝试安装可选包（如果存在）
sudo apt-get install -y python3-libnvinfer-plugin 2>/dev/null || echo "python3-libnvinfer-plugin 不存在，跳过"
sudo apt-get install -y libnvinfer-headers-dev 2>/dev/null || echo "libnvinfer-headers-dev 不存在，跳过"
sudo apt-get install -y libnvinfer-headers-plugin-dev 2>/dev/null || echo "libnvinfer-headers-plugin-dev 不存在，跳过"

# 尝试安装 UFF converter（可选，用于模型转换）
sudo apt-get install -y uff-converter-tf 2>/dev/null || echo "uff-converter-tf 不存在，跳过"
sudo apt-get install -y python3-uff 2>/dev/null || echo "python3-uff 不存在，跳过"
```

### 3.3 验证 TensorRT 安装

```bash
# 验证 TensorRT Python 绑定
python3 -c "import tensorrt as trt; print(f'TensorRT 版本: {trt.__version__}')"

# 预期输出:
# TensorRT 版本: 8.x.x

# 如果成功，继续验证其他功能
python3 -c "import tensorrt as trt; print(trt.get_version())"
```

**如果验证失败：**

```bash
# 检查 Python 路径
python3 -c "import sys; print('\n'.join(sys.path))"

# 检查 TensorRT 库路径
ldconfig -p | grep nvinfer

# 手动添加路径（如果需要）
export LD_LIBRARY_PATH=/usr/lib/aarch64-linux-gnu:$LD_LIBRARY_PATH
```

### 3.4 配置环境变量

```bash
# 添加到 ~/.bashrc
cat >> ~/.bashrc << 'EOF'

# TensorRT 环境变量
export LD_LIBRARY_PATH=/usr/lib/aarch64-linux-gnu:$LD_LIBRARY_PATH
export PATH=/usr/local/cuda/bin:$PATH
export CUDA_HOME=/usr/local/cuda
EOF

# 使环境变量生效
source ~/.bashrc
```

## 4. 项目部署

### 4.1 准备 Swap 空间（强烈推荐）

**Jetson Nano 只有 4GB 内存，必须增加 Swap！**

```bash
# 检查当前 Swap
sudo swapon --show

# 如果没有 Swap 或 Swap 不足 4GB，创建 Swap 文件
sudo fallocate -l 4G /var/swapfile

# 设置权限
sudo chmod 600 /var/swapfile

# 格式化为 Swap
sudo mkswap /var/swapfile

# 启用 Swap
sudo swapon /var/swapfile

# 验证 Swap
free -h

# 设置开机自动挂载
echo '/var/swapfile swap swap defaults 0 0' | sudo tee -a /etc/fstab
```

### 4.2 获取项目源码

```bash
# 创建工作目录
mkdir -p ~/projects
cd ~/projects

# 克隆项目（如果使用 Git）
git clone https://github.com/LeighRichard/camena-control.git

# 或解压项目压缩包（如果使用压缩包）
# unzip camena-control.zip

# 进入项目目录
cd camena-control/jetson
```

### 4.3 运行部署脚本

**推荐使用非交互模式：**

```bash
# 添加执行权限
chmod +x deploy.sh

# 运行部署脚本（非交互模式，跳过相机 SDK）
./deploy.sh -y
```

5. ✅ 安装系统依赖
6. ✅ 创建虚拟环境

## 5. 相机 SDK 安装

### 5.1 Intel RealSense SDK 安装

**方案 1: 尝试 pip 安装（简单但可能失败）**

```bash
# 激活虚拟环境
source venv/bin/activate

# 尝试安装
pip install pyrealsense2

# 验证安装
python -c "import pyrealsense2 as rs; print(f'RealSense SDK: {rs.__version__}')"
```

**方案 2: 从源码编译（推荐，稳定）**

```bash
# 1. 安装依赖
sudo apt-get install -y \
    libusb-1.0-0-dev \
    pkg-config \
    libgtk-3-dev \
    libglfw3-dev \
    libgl1-mesa-dev \
    libglu1-mesa-dev \
    cmake

# 2. 克隆源码（使用支持 Python 3.6 的旧版本）
cd ~/projects
git clone https://github.com/IntelRealSense/librealsense.git
cd librealsense

# 切换到支持 Python 3.6 的版本（v2.50.0）
git checkout v2.50.0

# 3. 创建构建目录
mkdir build && cd build

# 4. 配置 CMake（指定 Python 3.6）
# 注意：v2.50.0 使用旧版 pybind11，支持 Python 3.6
cmake .. \
    -DBUILD_PYTHON_BINDINGS=bool:true \
    -DPYTHON_EXECUTABLE=/usr/bin/python3.6 \
    -DCMAKE_BUILD_TYPE=Release \
    -DBUILD_EXAMPLES=false \
    -DBUILD_TESTS=false

# 5. 编译（使用 2 个核心避免内存不足）
make -j2

# 6. 安装
sudo make install

# 7. 更新库缓存
sudo ldconfig

# 8. 验证安装
python3 -c "import pyrealsense2 as rs; print(f'RealSense SDK: {rs.__version__}')"
```

**重要说明：**
- ⚠️ **必须使用 v2.50.0 或更早版本**，因为新版本的 pybind11 要求 Python 3.7+
- ⚠️ 如果使用最新版本会遇到错误：`Could NOT find PythonInterp: Found unsuitable version "3.6.9"`
- ✅ v2.50.0 是最后一个支持 Python 3.6 的稳定版本

**方案 3: 使用预编译包（如果可用）**

```bash
# 下载预编译包（需要找到合适的版本）
wget https://example.com/pyrealsense2-2.50.0-cp36-cp36m-linux_aarch64.whl

# 安装
pip install pyrealsense2-2.50.0-cp36-cp36m-linux_aarch64.whl
```

## 8. 故障排除

### 8.1 TensorRT 相关问题

**问题 1: 无法定位软件包 python3-libnvinfer-plugin**

**原因：** JetPack 4.6 中某些 TensorRT 包名称不同或不存在。

**解决方案：**
```bash
# 只安装核心包即可
sudo apt-get install -y \
    python3-libnvinfer \
    python3-libnvinfer-dev \
    libnvinfer-bin \
    libnvinfer-dev \
    libnvinfer-plugin8

# 验证安装
python3 -c "import tensorrt as trt; print(f'TensorRT 版本: {trt.__version__}')"
```

**注意：** 以下包在 JetPack 4.6 中可能不存在，可以安全跳过：
- `python3-libnvinfer-plugin`
- `libnvinfer-headers-dev`
- `libnvinfer-headers-plugin-dev`
- `uff-converter-tf`
- `python3-uff`

这些包是可选的，不影响核心功能。

**问题 2: ModuleNotFoundError: No module named 'tensorrt'**

```bash
# 解决方案：安装 TensorRT Python 绑定
sudo apt-get install -y python3-libnvinfer python3-libnvinfer-dev

# 验证
python3 -c "import tensorrt; print(tensorrt.__version__)"
```

**问题 3: TensorRT 版本不匹配**

```bash
# 检查版本
dpkg -l | grep libnvinfer
python3 -c "import tensorrt; print(tensorrt.__version__)"

# 重新安装匹配版本
sudo apt-get install --reinstall libnvinfer8=8.x.x-1+cuda10.2
```

**问题 4: CUDA 相关错误**

```bash
# 检查 CUDA 安装
nvcc --version
ls -la /usr/local/cuda

# 设置环境变量
export PATH=/usr/local/cuda/bin:$PATH
export LD_LIBRARY_PATH=/usr/local/cuda/lib64:$LD_LIBRARY_PATH
```

### 8.2 相机 SDK 相关问题

**问题 1: librealsense 编译失败 - Python 版本不匹配**

**错误信息：**
```
Could NOT find PythonInterp: Found unsuitable version "3.6.9",
but required is at least "3.7"
```

**原因：** 最新版本的 librealsense 使用 pybind11 2.13.6，要求 Python 3.7+，但 Jetson Nano 默认是 Python 3.6。

**解决方案：**
```bash
# 使用支持 Python 3.6 的旧版本
cd ~/projects/librealsense
git checkout v2.50.0

# 重新编译
mkdir build && cd build
cmake .. \
    -DBUILD_PYTHON_BINDINGS=bool:true \
    -DPYTHON_EXECUTABLE=/usr/bin/python3.6 \
    -DCMAKE_BUILD_TYPE=Release \
    -DBUILD_EXAMPLES=false \
    -DBUILD_TESTS=false

make -j2
sudo make install
sudo ldconfig
```

**重要：**
- ✅ v2.50.0 是最后一个支持 Python 3.6 的稳定版本
- ⚠️ 不要使用 master 分支或 v2.51.0+ 版本
- ✅ v2.50.0 功能完整，足够用于生产环境

**问题 2: pyrealsense2 安装失败**

```bash
# 解决方案：从源码编译
# 参考: 第 5.1 节

# 或使用 Docker
docker pull intelrealsense/realsense-ros
```

**问题 3: 相机设备无法识别**

```bash
sudo tee /etc/udev/rules.d/99-realsense-libusb.rules << 'EOF'
SUBSYSTEMS=="usb", ATTRS{idVendor}=="8086", ATTRS{idProduct}=="0b3a", MODE:="0666", GROUP:="plugdev"
SUBSYSTEMS=="usb", ATTRS{idVendor}=="8086", ATTRS{idProduct}=="0b07", MODE:="0666", GROUP:="plugdev"
EOF

# 重新加载 udev 规则
sudo udevadm control --reload-rules
sudo udevadm trigger
```
