## 项目概述

### 系统简介

本系统是一个基于 **NVIDIA Jetson Nano** 和 **STM32F407** 的专业级嵌入式相机位置控制与自动拍摄系统。系统采用 **Intel RealSense D415** 深度相机进行图像采集，通过深度学习实现农作物目标检测，利用云台和滑轨实现相机位置的精确控制，支持完整的自动拍摄流程。

### 应用场景

- 🌾 **农业监测**: 农作物生长监测、病虫害检测
- 🏭 **工业检测**: 产品质量检测、缺陷识别
- 🔬 **科研实验**: 多角度图像采集、3D 重建
- 📸 **专业摄影**: 自动化产品拍摄、全景拍摄

### 核心特性

- ✅ **智能目标检测** - YOLOv5/YOLOv8 + TensorRT GPU 加速
- ✅ **精确运动控制** - PID 控制 + S 曲线速度规划
- ✅ **自动拍摄流程** - 路径规划 + 时序控制 + 振动衰减
- ✅ **多端远程控制** - Web + 手机 APP + 桌面 APP
- ✅ **远程网络访问** - WiFi 热点 + 4G/5G 网络
- ✅ **安全保护机制** - 限位检测 + 急停 + 看门狗
- ✅ **实时视频预览** - MJPEG 流 + 目标检测叠加
- ✅ **数据管理** - 图像存储 + 历史记录 + 批量导出

## 技术路线

### 整体技术路线

本项目采用 **双控制器分层架构** + **多端协同** 的技术路线：

```
┌─────────────────────────────────────────────────────────────┐
│                    技术路线总览                              │
└─────────────────────────────────────────────────────────────┘

第一层：硬件执行层 (STM32)
  └─> 实时运动控制、安全监控、电机驱动

第二层：智能控制层 (Jetson Nano)
  └─> 相机控制、目标检测、任务调度、通信管理

第三层：网络服务层 (Jetson Nano)
  └─> Web 服务器、视频流、远程访问、数据管理

第四层：用户交互层 (多端客户端)
  └─> Web 界面、手机 APP、桌面 APP
```

### 技术选型思路

#### 1. 主控制器选择：NVIDIA Jetson Nano

**选型理由**:
- ✅ **GPU 加速**: 128 核 Maxwell GPU，支持 CUDA 和 TensorRT
- ✅ **深度学习**: 原生支持 PyTorch、TensorFlow、ONNX
- ✅ **视觉处理**: 硬件加速的图像处理和编解码
- ✅ **接口丰富**: USB 3.0、UART、GPIO、CSI 摄像头接口
- ✅ **生态完善**: JetPack SDK、大量开源项目支持
- ✅ **功耗适中**: 5-10W，适合嵌入式部署

**对比其他方案**:
- vs 树莓派 4: GPU 性能强 10 倍以上，深度学习推理快 5-10 倍
- vs x86 工控机: 功耗低 5-10 倍，成本低 3-5 倍
- vs 纯 STM32: 可运行 Linux 系统，支持 Python 开发

#### 2. 运动控制器选择：STM32F407

**选型理由**:
- ✅ **实时性强**: 168MHz Cortex-M4，无操作系统抖动
- ✅ **定时器丰富**: 14 个定时器，支持 PWM 和编码器接口
- ✅ **浮点运算**: 硬件 FPU，适合 PID 计算
- ✅ **通信接口**: 多路 UART、SPI、I2C
- ✅ **成本低**: 开发板 50-100 元
- ✅ **生态成熟**: HAL 库、大量示例代码

**对比其他方案**:
- vs Arduino: 性能强 10 倍，定时器更多
- vs ESP32: 实时性更好，定时器精度更高
- vs 纯 Jetson GPIO: 实时性强 100 倍以上


#### 3. 相机选择：Intel RealSense D415

**选型理由**:
- ✅ **RGB + 深度**: 同时获取彩色图像和深度信息
- ✅ **高精度**: 深度精度 < 2%，适合距离测量
- ✅ **SDK 完善**: librealsense 支持 Python/C++
- ✅ **USB 3.0**: 高带宽，支持高分辨率实时传输
- ✅ **室外适用**: 主动红外，不受环境光影响

**对比其他方案**:
- vs 普通 USB 摄像头: 增加深度信息，可测距
- vs 双目相机: 精度更高，SDK 更完善
- vs Kinect: 体积小，功耗低，USB 供电

#### 4. 深度学习框架：TensorRT + ONNX

**选型理由**:
- ✅ **NVIDIA 优化**: 针对 Jetson 平台深度优化
- ✅ **推理速度快**: 比 PyTorch 快 3-5 倍
- ✅ **模型通用**: 支持 ONNX 格式，兼容各种训练框架
- ✅ **精度可选**: FP32/FP16/INT8 量化
- ✅ **内存优化**: 显存占用小

**工作流程**:
```
PC 训练 (PyTorch/TensorFlow)
  ↓
导出 ONNX 模型
  ↓
Jetson Nano 转换为 TensorRT
  ↓
实时推理 (15-25 FPS)
```

#### 5. 通信协议：UART + 自定义协议

**选型理由**:
- ✅ **简单可靠**: 硬件支持好，调试方便
- ✅ **实时性好**: 115200 bps 足够运动控制
- ✅ **抗干扰**: CRC16 校验，重传机制
- ✅ **低成本**: 只需 3 根线 (TX/RX/GND)

**协议设计**:
```
帧格式: [HEAD][LEN][CMD][DATA][CRC16][TAIL]
校验: CRC16-CCITT
超时: 100ms
重试: 最多 3 次
```

#### 6. 多端开发框架

| 平台 | 框架 | 选型理由 |
|------|------|----------|
| Web | Flask + JavaScript | 轻量级，易部署，跨平台 |
| 手机 | Flutter | 一套代码，iOS/Android 双平台 |
| 桌面 | Electron + React | 跨平台，Web 技术栈，开发效率高 |

### 通信协议设计

#### 帧格式

```
┌──────┬──────┬──────┬──────────┬──────────┬──────┐
│ HEAD │ LEN  │ CMD  │  DATA    │ CHECKSUM │ TAIL │
│ 0xAA │ 1B   │ 1B   │  N Bytes │   2B     │ 0x55 │
└──────┴──────┴──────┴──────────┴──────────┴──────┘
```

#### 指令类型

| CMD | 名称 | DATA 格式 | 说明 |
|-----|------|-----------|------|
| 0x01 | 位置控制 | axis(1B) + target(4B) | 设置目标位置 |
| 0x02 | 状态查询 | 无 | 查询当前状态 |
| 0x03 | 参数配置 | param_id(1B) + value(4B) | 配置参数 |
| 0x04 | 急停 | 无 | 紧急停止 |
| 0x05 | 归零 | axis(1B) | 归零校准 |

#### 响应类型

| CMD | 名称 | DATA 格式 | 说明 |
|-----|------|-----------|------|
| 0x81 | 位置响应 | status(1B) | 位置设置结果 |
| 0x82 | 状态响应 | pan(4B)+tilt(4B)+rail(4B)+status(1B) | 当前状态 |
| 0x83 | 配置响应 | status(1B) | 配置结果 |
| 0x84 | 急停响应 | status(1B) | 急停结果 |
| 0x85 | 归零响应 | status(1B) | 归零结果 |

#### 校验机制

- **算法**: CRC16-CCITT
- **多项式**: 0x1021
- **初始值**: 0xFFFF
- **范围**: HEAD 到 DATA 的所有字节

#### 超时与重试

- **超时时间**: 100ms
- **重试次数**: 最多 3 次
- **失败处理**: 返回错误，触发看门狗

## 创新点

### 1. 双控制器分层架构 🏆

**创新描述**:
将智能控制和实时运动控制分离，Jetson Nano 负责高级逻辑和深度学习，STM32 负责实时运动控制。

**技术优势**:
- ✅ **实时性保证**: STM32 无操作系统抖动，运动控制精度高
- ✅ **智能化**: Jetson Nano GPU 加速深度学习推理
- ✅ **可靠性**: 双控制器互相监控，通信看门狗
- ✅ **可扩展**: 各层独立开发，易于升级

**对比传统方案**:
- vs 单片机方案: 增加深度学习能力
- vs 纯 PC 方案: 实时性提升 100 倍，功耗降低 10 倍
- vs 树莓派方案: GPU 性能强 10 倍

### 2. 滚动快门时序控制 🏆

**创新描述**:
针对 D415 相机的滚动快门特性，设计了完整的时序控制流程：运动停止 → 振动衰减 → 多帧稳定 → 拍摄确认。

**技术细节**:
```python
# 拍摄时序控制
1. 发送运动指令
2. 等待运动完成信号
3. 检测振动衰减 (settle_time)
4. 等待多帧稳定 (capture_frames)
5. 触发拍摄
6. 确认拍摄完成
7. 允许下一次运动
```

**解决的问题**:
- ❌ 传统方案: 运动中拍摄 → 图像模糊
- ✅ 本方案: 完全静止后拍摄 → 图像清晰

### 故障排除

#### 常见问题

**1. Jetson Nano 无法启动**
- 检查电源是否使用桶形接口 (不能用 Micro-USB)
- 确认电源适配器为 5V 4A
- 检查 microSD 卡是否正确烧录系统

**2. 相机无法识别**
```bash
# 检查相机连接
lsusb | grep Intel
# 应该看到 Intel RealSense D415

# 测试相机
realsense-viewer

# 检查权限
sudo chmod 666 /dev/video*
```

**3. 串口通信失败**
```bash
# 检查串口设备
ls -l /dev/ttyTHS*

# 添加用户到 dialout 组
sudo usermod -a -G dialout $USER

# 测试串口
python3 -m serial.tools.miniterm /dev/ttyTHS1 115200
```

**4. 深度学习推理慢**
- 确认使用 TensorRT 而非 PyTorch
- 检查 GPU 是否正常工作: `tegrastats`
- 降低输入分辨率或使用更小的模型

**5. 视频流卡顿**
- 检查网络带宽
- 降低视频分辨率或帧率
- 关闭不必要的后台程序

**6. 电机不转**
- 检查电源连接 (24V)
- 检查驱动器使能信号
- 检查步进脉冲输出
- 测量驱动器输出电压

**7. 限位开关不工作**
- 检查开关接线
- 测量开关触发时的电平
- 确认 STM32 引脚配置为上拉输入
